# Peter Fintl — Executive Media Hub

A polished executive media hub hosted on **GitHub Pages** at `https://fintlp.github.io/latest-news/`. Driven entirely by static JSON files with no build step. Sections include curated media commentary, video highlights, publications, speaking engagements, a LinkedIn Posts wall, and an auto-updated news feed.

## LinkedIn Posts section

### Overview

`data/linkedin-posts.json` contains 582 posts exported from **niomaker** (a LinkedIn data export tool). The file is generated by `scripts/parse-linkedin-csv.js` from a CSV export and then enriched by four download scripts. All media is stored locally to avoid LinkedIn CDN token expiry (tokens expire every ~6 weeks).

### Post data shape

```json
{
  "id": "7297880097703759872",
  "permalink": "https://www.linkedin.com/posts/...",
  "title": "First line of the post (≤120 chars)",
  "text": "Full post text",
  "topics": ["China & EV", "ADAS & Autonomous"],
  "imageUrl": "assets/linkedin-images/7297880097703759872-0.jpg",
  "imageUrls": ["assets/linkedin-images/7297880097703759872-0.jpg"],
  "videoUrl": "assets/linkedin-videos/7297880097703759872.mp4",
  "documentUrl": "assets/linkedin-docs/7297880097703759872.pdf",
  "documentThumb": "assets/linkedin-docs/7297880097703759872-thumb.jpg",
  "articleUrl": "https://handelsblatt.com/...",
  "likes": 142,
  "comments": 18,
  "shares": 4,
  "engagement": 232,
  "publishDate": "2026-04-10",
  "source": "LinkedIn",
  "author": "Peter Fintl"
}
```

**`imageUrl` priority** (what ends up as the card image):

| Source | Condition |
|---|---|
| `assets/linkedin-images/` | Post had its own attached photo |
| `assets/linkedin-videos/{id}-thumb.jpg` | Video post, no other image |
| `assets/linkedin-docs/{id}-thumb.jpg` | PDF post, no other image |
| `assets/linkedin-articles/{id}-article.*` | Article-promotion post (no LinkedIn photo) |
| `null` | Text-only post → LinkedIn-branded placeholder shown |

**Current counts:**
- 417 posts → `linkedin-images`
- 30 posts → `linkedin-videos` thumbnail
- 6 posts → `linkedin-docs` thumbnail
- 41 posts → `linkedin-articles` OG image
- 88 posts → no image (text-only placeholder)
- 271 posts have `articleUrl` (the "Read Article" button appears in the modal)
- 30 posts have `videoUrl` (local MP4 via LFS)
- 6 posts have `documentUrl` (local PDF with download button)

### Updating LinkedIn posts (re-export flow)

Run these steps whenever you export a new CSV from niomaker:

```bash
# 1. Place the new CSV in the repo root as LI_POSTS_fintlp_568.csv
#    (overwrite the old file)

# 2. Regenerate JSON (idempotent — preserves all local asset paths)
node scripts/parse-linkedin-csv.js

# 3. Download any new images (skips already-downloaded)
node scripts/download-linkedin-images.js

# 4. Download any new PDFs + thumbnails
node scripts/download-linkedin-docs.js

# 5. Fetch OG images for new article-promotion posts
node scripts/download-linkedin-article-images.js

# 6. Download any new videos (requires Git LFS + ffmpeg)
node scripts/download-linkedin-videos.js

# 7. Optionally resolve lnkd.in shortlinks to real article URLs
node scripts/resolve-article-urls.js
```

All scripts are **idempotent** — already-processed posts are skipped. Run them in order after a re-export.

**Prerequisites for step 6:**
```bash
brew install git-lfs ffmpeg
git lfs install
```

### Why local assets?

LinkedIn CDN URLs contain a signed expiry token (`?e=<unix_timestamp>`). Tokens typically expire every 6 weeks. After expiry all post images return 403. By downloading all media locally and pointing `imageUrl` to `assets/linkedin-*`, the site never needs CDN token renewal.

### parse-linkedin-csv.js — merge functions

The parser calls four merge functions in sequence after building posts from the CSV. These preserve local asset paths across re-exports:

| Function | Reads from | Sets |
|---|---|---|
| `mergeLocalImages()` | `assets/linkedin-images/{id}-{n}.{ext}` | `imageUrl`, `imageUrls` |
| `mergeLocalDocs()` | `assets/linkedin-docs/{id}.pdf` + `{id}-thumb.*` | `documentUrl`, `documentThumb`, `imageUrl` |
| `mergeLocalArticleImages()` | `assets/linkedin-articles/{id}-article.*` + existing JSON for `articleUrl` | `articleUrl`, `imageUrl` (only if no other image) |
| `mergeLocalVideos()` | `assets/linkedin-videos/{id}.mp4` + `{id}-thumb.*` | `videoUrl`, `imageUrl` (only if no other image) |

`mergeLocalArticleImages` also reads the existing `data/linkedin-posts.json` to re-apply any `articleUrl` values from the previous run (including manually set ones).

### Video hosting — local vs. production

GitHub Pages does **not** serve Git LFS files (it delivers the 134-byte pointer instead of the binary). Videos are therefore handled differently by environment:

- **localhost / 127.0.0.1**: inline `<video>` player with the local MP4
- **Production** (`fintlp.github.io`): thumbnail image + "Watch on LinkedIn →" overlay link to the post permalink

Detection in `app.js`:
```js
const isLocal = location.hostname === 'localhost' || location.hostname === '127.0.0.1';
```

### articleUrl and the "Read Article" button

When a post references an external article, `articleUrl` stores the link and a "📰 Read Article" button appears in the post modal. 271 posts currently have `articleUrl`. Many are `lnkd.in` shortlinks — these work correctly in a browser (they redirect to the article) even though they cannot be resolved server-side (lnkd.in returns HTTP 200 with JavaScript redirect, not a 3xx).

To attempt re-resolution of shortlinks:
```bash
node scripts/resolve-article-urls.js
```

### UI behaviour

The post cards show:
- **Photo posts**: the image with no badge
- **PDF posts**: the PDF thumbnail with a `📄 PDF` badge (top-right corner)
- **Video posts**: the video thumbnail with a `▶` play badge (bottom-right corner)
- **Article-promo posts**: the article OG image
- **Text-only posts**: LinkedIn blue gradient placeholder with a subtle "in" watermark

Clicking a card opens a modal with the full text, engagement stats, and action buttons:
- "📰 Read Article" — visible only when `articleUrl` is set
- "📄 Download PDF" — visible only when `documentUrl` is set
- "View on LinkedIn →" — always visible

Both the article and PDF buttons use `style.display` (not the `hidden` attribute) because CSS `.btn { display: … }` rules can override the `hidden` attribute.

## News feed

### How it works

1. **GitHub Actions** runs `scripts/fetch-news.js` daily at 09:11 Vienna time.
2. The script queries **Google News RSS** (13 locales) for `"Peter Fintl"` and optionally **Brave Search** for topic queries.
3. Results are normalised, deduped, and merged into `data/archive.json` (rolling 1-year window).
4. On page load, `app.js` fetches `data/archive.json` and renders the **Latest Coverage** section.

### Data sources

| Source | What it covers |
|---|---|
| Google News RSS — 13 locales | en-US, en-GB, en-AU, de-AT, de-DE, de-CH, zh-TW, zh-CN, ja, ko, fr-FR, it-IT, nl-NL |
| Brave Search News API | Optional topic queries; requires `BRAVE_API_KEY` secret |

### Image pipeline

Each news card tries the following in order:

| Priority | Source |
|---|---|
| 1 | `og:image` / `twitter:image` scraped from article page |
| 2 | RSS `<media:content>` / `<media:thumbnail>` / `<enclosure>` |
| 3 | Outlet logo matched via `data/as-seen-in.json` (supports `aliases`) |
| 4 | `EXTRA_SOURCE_DOMAINS` map in `fetch-news.js` |
| 5 | Domain extracted from source name |
| 6 | `google.com/s2/favicons?domain={host}&sz=256` (always resolves) |

Cards whose `imageUrl` is a favicon URL get `object-fit: contain` with padding in CSS.

### Adding articles manually

Add an entry to `data/manual-overrides.json`:

```json
{
  "title": "Article title",
  "url": "https://www.outlet.com/article-path",
  "source": "Outlet Name",
  "sourceUrl": "https://www.outlet.com",
  "faviconUrl": "https://www.google.com/s2/favicons?domain=outlet.com&sz=128",
  "imageUrl": null,
  "publishedAt": "2026-04-29T00:00:00.000Z",
  "snippet": "Short description.",
  "manually_added": true
}
```

Then rebuild locally:
```bash
node scripts/fetch-news.js
git add data/manual-overrides.json data/archive.json data/news.json
git commit -m "feat: add article to manual-overrides"
git push
```

**Important:** Use plain ASCII double quotes only. Typographic quotes (`"` `"` `„`) silently break JSON parsing.

### Adding outlet logos

Add an entry to `data/as-seen-in.json`:
```json
{
  "name": "Outlet Name",
  "aliases": ["shortname", "shortname.com"],
  "url": "https://www.outlet.com/"
}
```

For frequent outlets not in `as-seen-in.json`, add them to `EXTRA_SOURCE_DOMAINS` in `scripts/fetch-news.js`:
```js
const EXTRA_SOURCE_DOMAINS = {
  'tablebriefings': 'table.media',
  // normalizeSourceKey(sourceName) → domain
};
```

## Deployment

```bash
git add -A
git commit -m "feat: ..."
git push
```

GitHub Pages serves from `main` branch root. The Actions workflow commits data changes automatically with `[skip ci]`.

**First-time setup:** Settings → Pages → Deploy from branch → `main` → `/` (root).
